## 简介
本文件系统性梳理红书项目的 Taro 多端构建配置体系，覆盖开发与生产环境的差异化策略、Babel 转译对 TypeScript、JSX、装饰器等语法的支持、多平台（微信小程序、H5、React Native）构建参数差异、代码分割与 Tree Shaking 的可配置点、以及热重载、Source Map、压缩混淆等开发体验相关配置。同时提供构建性能优化建议与常见问题的解决方案。

## 项目结构
红书项目采用 Taro 4 + React + Webpack5 的多端统一工程化方案，核心配置集中在 config 目录下，按环境拆分基础配置与差异化配置；TypeScript 编译与 Babel 转译分别由 tsconfig.json 与 babel.config.js 驱动；页面级配置通过各页面的 index.config.ts 进行声明式控制。

- 统一配置入口：在 config/index.ts 中定义基础配置与环境合并逻辑，区分 development 与 production。

### 多平台构建参数差异
- 微信小程序（mini）
  - pxtransform：启用 px → rpx 转换，适配小程序设计规范。
  - CSS Modules：启用模块化命名，避免样式冲突。
- H5（h5）
  - autoprefixer：自动添加浏览器前缀，提升兼容性。
  - CSS Modules：同上，模块化命名策略一致。
  - publicPath 与 staticDirectory：控制资源路径与静态资源目录。
  - MiniCssExtractPlugin：忽略顺序警告，分离与缓存 CSS。
- React Native（rn）
  - appName：RN 应用名称。
  - postcss.cssModules：关闭 CSS Modules，避免不必要开销。

### 代码分割、懒加载与 Tree Shaking
- 代码分割与懒加载
  - 在 Taro 项目中，可通过路由级或组件级动态导入实现按需加载，减少首屏体积。
  - 建议结合平台特性（如小程序分包）进一步优化启动速度。
- Tree Shaking
  - 确保模块以 ES Module 形式导出，避免副作用（sideEffects）污染，提高摇树效果。
  - 生产构建通常默认开启作用域提升与死代码消除，配合合理的模块组织可获得更好收益。

## 性能考虑
- 构建缓存：当前基础配置禁用了缓存，建议在 CI 或本地开发中评估开启缓存以缩短二次构建时间。
- 体积分析：生产配置预留了 webpackChain 扩展点，可接入体积分析插件，定位大体积依赖与重复模块。
- 预渲染：可按需启用预渲染插件，改善 H5 首屏加载表现。
- 依赖优化：保持依赖精简，优先使用按需引入与动态导入，减少不必要的打包体积。
- 平台优化：小程序端充分利用分包与资源压缩；H5 端关注首屏 CSS 与 JS 的分离与缓存策略。

## 故障排查指南
- 开发代理无效
  - 检查开发配置中的 devServer.proxy 是否正确指向 API 基础地址；确认环境变量 TARO_APP_API_BASE_URL 是否设置。
- TypeScript 装饰器报错
  - 确认 tsconfig.json 已启用 experimentalDecorators；Babel 装饰器插件版本与预设是否匹配。
- H5 样式模块化异常
  - 检查 CSS Modules 命名模式与生成规则是否一致；确认 scoped 名称冲突。
- 生产构建体积过大
  - 使用 webpackChain 扩展点接入体积分析插件，定位冗余模块；结合动态导入与分包策略优化。
- React Refresh 不生效
  - 确认开发脚本使用 watch 模式；检查相关插件版本与配置。
