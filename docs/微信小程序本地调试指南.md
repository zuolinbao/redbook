# 微信小程序本地调试指南

本文档介绍如何使用 Proxyman 实现微信小程序的本地调试，将线上 API 请求映射到本地后端服务。

## 1. 调试原理

```
微信小程序
    │
    │  wx.request('https://yipin.ctwing.cn/v2/api/xxx')
    ▼
Proxyman (HTTP 代理)
    │
    ├─ 远程映射规则: `https://yipin.ctwing.cn` → `http://10.251.23.207:30177`
    │
    ├─ Script 脚本: 改写 Origin 为 `http://localhost:30177`
    │
    ▼
本地后端服务 (`http://localhost:30177`)
```

## 2. 环境配置

本项目使用 Taro 多端框架，环境变量配置如下：

### 2.1 开发环境配置 (.env.development)

```bash
# API 基础地址配置（不包含 /cmp-api）
TARO_APP_API_BASE_URL=http://10.251.23.207:30177

# 微信小程序开发环境 API 地址
TARO_APP_WEAPP_API_BASE_URL=https://yipin.ctwing.cn

# H5 环境代理前缀
TARO_APP_H5_PROXY_PREFIX=/cmp-api
```

### 2.2 HTTP 请求模块 (src/utils/http.ts)

微信小程序环境下，请求会自动使用 `TARO_APP_WEAPP_API_BASE_URL`：

```typescript
// 微信小程序环境：开发使用测试地址，打包使用生产地址
if (isWeapp) {
  return process.env.TARO_APP_WEAPP_API_BASE_URL || ''
}
```

## 3. Proxyman 配置步骤

### 3.1 远程映射规则配置

在 Proxyman 中创建远程映射规则（参考截图）：

![远程映射规则配置](/docs/images/proxyman-mapping.png)

| 配置项 | 值 |
|--------|-----|
| 名称 | `cmp` |
| 规则 | `https://yipin.ctwing.cn` |
| 匹配模式 | `ANY` (匹配所有路径) |
| 包含子路径 | 不勾选 |

**映射目标设置：**

| 配置项 | 值 |
|--------|-----|
| 协议 | `http` |
| 主机 | `10.251.23.207` |
| 端口 | `30177` |
| 路径 | 空 |

**高级选项：**

| 配置项 | 状态 |
|--------|------|
| 匹配后保留原始网址 | 不勾选 |
| 保留主机关头 | 不勾选 |

### 3.2 Origin 重写脚本配置

在 Proxyman 的 Script Editor 中添加脚本（参考截图）：

![Script 脚本配置](/docs/images/proxyman-script.png)

```javascript
function onRequest(context, url, request) {
  request.headers['Origin'] = 'http://localhost:30177';
  return request;
}
```

**配置说明：**

| 配置项 | 状态 |
|--------|------|
| Name | `cmp` |
| URL | `https://yipin.ctwing.cn` |
| Matching Mode | `ANY` |
| Include Subpaths | 不勾选 |
| Running Script On | ✅ Request |

## 4. 微信开发者工具配置

### 4.1 开启不校验合法域名

在微信开发者工具中，进入 **项目配置** → **本地设置**：

- ✅ 勾选 **不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书**

> 注意：此选项仅在开发阶段开启，生产环境必须使用已配置的合法域名。

### 4.2 启动开发服务器

确保本地后端服务已启动：

```bash
# 启动后端服务 (端口: 30177)
pnpm dev:h5
```

### 4.3 启动小程序开发

```bash
# 微信小程序开发模式
pnpm dev:weapp
```

## 5. 调试流程

1. 启动本地后端服务（端口 30177）
2. 启动微信开发者工具，导入项目
3. 开启微信开发者工具的「不校验合法域名」选项
4. 确保 Proxyman 正常运行且规则已激活
5. 在小程序中触发 API 请求
6. 请求流程：
   - 小程序发起请求到 `https://yipin.ctwing.cn/xxx`
   - Proxyman 拦截并映射到 `http://10.251.23.207:30177/xxx`
   - Script 脚本修改 Origin 为 `http://localhost:30177`
   - 本地后端收到请求并返回响应
   - 响应通过 Proxyman 返回给小程序

## 6. 常见问题

### Q1: 请求失败，提示域名不在合法域名列表中

**解决方案：** 在微信开发者工具中勾选「不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书」

### Q2: 后端服务无法收到请求

**解决方案：**
1. 确认本地后端服务已启动
2. 检查 Proxyman 的远程映射规则是否正确
3. 确认手机/模拟器已配置 Proxyman 代理

### Q3: Origin 校验失败

**解决方案：**
1. 确保 Script 脚本已保存并激活
2. 检查脚本中 Origin 值是否与后端服务地址一致
3. 确认后端服务配置允许该 Origin

### Q4: 如何确认请求是否被 Proxyman 拦截？

**解决方案：** 在 Proxyman 中查看请求列表，确认请求是否显示「cmp」规则标签

## 7. 生产环境部署

当需要发布到生产环境时：

1. 将 `.env.production` 中的 API 地址配置为生产域名
2. 确保所有 API 请求使用 HTTPS
3. 在微信公众平台配置合法域名
4. 移除开发调试代码和临时配置

```bash
# 构建生产版本
pnpm build:weapp
```
